# Conductor Development Workflow

## 1. Tracking
- Development is organized into **tracks**.
- Each track has a `spec.md` and a `plan.md`.

## 2. Planning
- Requirements must be stabilized in `spec.md` before implementation.
- `plan.md` outlines the implementation steps.

## 3. Implementation
- Follow Test-Driven Development (TDD) where possible.
- Required test coverage: >80%.
- Commit changes frequently (per task).

## 4. Verification
- All tests must pass before completing a track.
- Phase completion requires manual verification.
